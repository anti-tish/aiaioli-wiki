# Code Should Own Code

`Concept` · `knowledge_systems` · *hypothesis* · visibility: **publishable**

'Why should code not be the responsibility of the computers that are going to interpret and execute that command anyway?' An inversion of the human-ownership-of-code assumption: if the machine runs it, perhaps the machine should write it, in the same way that you don't write in the hardware register of the CPU. Not an AI-takes-over argument — a logical-division argument. Personal computers are less than 100 years old; the assumption that humans write code is a historical contingency, not a fundamental truth.
