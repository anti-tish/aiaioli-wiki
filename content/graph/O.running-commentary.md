# The Running Commentary Problem

`Observation` · `capture` · *personal_truth* · visibility: **publishable**

The unmet need underneath every tool she's tried, finally named: 'none of these capture THE RUNNING COMMENTARY of a brain.' Diarly is 'the only thing that seems to really capture the internal monologue running commentary aspect of a brain like nothing else' — and it's 'kind of limited, really just a journal app.' The whole search (Airtable, SQLite, Obsidian, fabric) circles this one gap: the facts are easy to store, the live monologue is the thing no schema holds. This is the founding problem the later project is built to answer.
