# Paradigms Outlast Constraints

`Observation` · `AI` · *hypothesis* · visibility: **publishable**

The prompt engineering paradigm made complete sense when it was built — hardware and software developing in tandem, humans doing translation because the machine couldn't meet them. But a hefty chunk of modern computing isn't done at a desktop by an outlet anymore. The hardware assumption has shifted. The interaction design hasn't caught up. Why are we still stuck to that paradigm? Because paradigms are sticky even when the physical constraints that produced them have dissolved. The QWERTY keyboard is still here. The prompt box is QWERTY. The receiving-human architecture is a different paradigm: designed around how AI should work given where hardware actually is now.
