# Two Routes to the Same Dissolve

`Connection` · `writing` · *hypothesis* · visibility: **publishable**

**From:** [[C.because-chain]]
**To:** [[P.frictionless-capture]]
**Relationship:** is_structurally_identical_to

Both dissolve blank-doc anxiety, from different directions, via different mechanisms. C.because-chain is a rhetorical technique: force the connective tissue into the open before writing anything (I think X because Y because Z — ugly, not prose, but the logic has to be there). P.frictionless-capture is a tool-design principle: remove all structure requirements at capture so the thought gets out before it knows what it is. One operates at the level of the argument; one at the level of the tool. Same problem (structure imposed at the wrong moment kills authentic output), same diagnosis (the demand arrives before you can answer it), same solution structure (relocate the structuring step — to later in the process for capture, to explicitly before for rhetoric). Two different people solving the same problem in different rooms arrived at the same architecture.
